# 01. 空間探索框架：三柱架構的應用

> 本章旨在定義本書的技術底層。透過「邏輯、證據、實踐」三支柱的框架，解釋我們如何結合地理理論、結構化資料庫與實地走訪，建立一個可驗證、可持續進化的知識體系。

## 1.1 從邏輯、證據到實踐的知識循環

在探究台灣地景時，我們採納「三柱架構 (Triad Methodology)」作為核心治理邏輯：

1.  **邏輯 (The Logic - Book)**：定義地理理論、越嶺效率模型與空間拓樸規範。
2.  **證據 (The Evidence - DB)**：以 SQLite 為核心，儲存高精度 DTM 高程、幾何頂點與文獻對合數據。
3.  **實踐 (The Practice - Log)**：紀錄實地走訪、POC 驗證過程與 QGIS 動態渲染結果。

這三者形成一個閉環：邏輯引導數據採集，數據驗證實踐結果，實踐反饋修正原始邏輯。

## 1.2 物理真實的萃取：為什麼 DTM 是唯一的水文真相

台灣地景的核心幾何由以下要素構成：

*   **脊線 (Ridgelines)**：定義了流域的邊界與越嶺的障礙。
*   **河源 (River Sources)**：向源侵蝕的終點，往往是古道選擇鞍部的物理誘因。
*   **古道 (Trails)**：順應地形能量損耗最小化的路徑，是脊線與河源交互作用下的「文明連線」。

本書將透過量化指標（如 E-index），分析這些幾何要素如何跨越時空交織在一起。

## 1.3 文明錨點的尋尋：從 HGIS 百年地圖找回原始聚落

在掌握物理地景後，下一步是將歷史文獻錨定至現代坐標系統。透過 HGIS (歷史地理資訊系統) 的比對，我們將古地圖上的地名、駐在所遺跡與現代數位地形模型進行疊合，確保每一條古道不僅是數據上的連線，更是具備時空脈絡的歷史路徑。
